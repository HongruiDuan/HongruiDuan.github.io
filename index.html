在Linux下配置NS3网络仿真器步骤，遇到的Bug以及解决方法

Posted on 2019-07-04

安装NS3

Tips：在安装之前最好确定好自己的Linux版本，越新的linux版本对应越新的NS3的版本，同时如果是用虚拟机的话要运行NS3需要较大的RAM，我的18.04 3g内存在运行 Hello simulator时由于内存溢出报错
报错信息与解决办法

安装步骤

1.安装依赖库

在安装NS3前需要安装一些依赖库，在安装库之前可以先更新一下apt

sudo apt-get update

安装依赖库

sudo apt-get install gcc g++ python python-dev C++和python

sudo apt-get install mercurial NS3代码维护使用的源码版本控制管理系统

sudo apt-get install bzr 运行python绑定ns-3-dev需要bazaar这个组件

sudo apt-get install gdb valgrind 调试工具

sudo apt-get install gsl-bin libgsl-dev libgsl23 支持更多精确WIFI模块的GNU Scientific Library (GSL)

sudo apt-get install flex bison libfl-dev 仿真必需的词法分析器和语法分析生成器

sudo apt-get install g++-3.4 gcc-3.4 Network Simulation Cradle (nsc) stacks需要gcc-3.4

sudo apt-get install tcpdump 读取pcap的packet traces

sudo apt-get install sqlite sqlite3 libsqlite3-dev 支持统计特性的数据库软件

sudo apt-get install libxml2 libxml2-dev xml的配置存储软件

sudo apt-get install libgtk2.0-0 libgtk2.0-dev 基于GTK的配置系统

sudo apt-get install vtun lxc 在虚拟机and ns-3上测试

sudo apt-get install uncrustify 支持utils/check-style.py 代码风格检查程序

sudo apt-get install doxygen graphviz imagemagick 文档生成器

sudo apt-get install texlive texlive-extra-utils texlive-latex-extra

sudo apt-get install python-sphinx dia

sudo apt-get install python-pygraphviz python-kiwi python- - pygoocanvas libgoocanvas-dev Gustavo’s ns-3-pyviz的可视化软件

sudo apt-get install libboost-signals-dev libboost-filesystem-dev 支持openflow 模块

sudo apt-get install openmpi* 支持基于 MPI的分布式仿真

2.为NS3创建工程目录

mkdir NS3

cd NS3

wget http://www.nsnam.org/release/ns-allinone-3.28.tar.bz2 （注意NS3版本与linux版本）

tar xjf ns-allinone-3.28.tar.bz2

3.编译NS3

cd ns-allinone-3.28

./build.py

cd ns-3.28

./waf distclean （清除整个build目录）

./waf configure –build-profile=debug –enable-examples –enable-tests（打开debug并开启例子及帮助）

./waf build

4.测试

./test.py -c core

5.运行hello-simulator

./waf –run hello-simulator

网络服务提供商的角度

效益可以描述为能量的效益，能量为不可再生的消耗

能量消耗应该是一个重要的标准

网络服务提供商的目的是最大化他们的利润和能量利用效率

本文的贡献

1将服务提供商的效益通过利润和能量利用效率的折中来构建

​ 用户的效益函数通过两方面来衡量：

​ 1.用户吞吐量的满意程度

​ 2.丢包率

​ 3.支付的费用

2.在提出的效益函数基础上通过一个Stackelberg博弈解决了联合售价和负 载分配的问题。分为single-follower和multi-follower

3.通过一种拉格朗日算法来解决用户角度的凸优化问题。在服务提供商的角度，我们提出了一种反复迭代的自私的竞价算法。

通过设置不同的参数，在最优的角度下我们分析了用户和服务提供商的关系

论文-协作D2D网络中两阶段的联合带宽和多中继调度博弈

Posted on 2019-06-18

今天阅读的论文是协作D2D网络中两阶段的联合带宽和多中继调度博弈

如何激励 MMDs 出租空余的带宽和将 OMDs分配给MMDs

第一阶段

MMDs 之间进行竞争 要素为共享的带宽和售价

第二阶段

OMDs 选择 MMDs 通过一个envolutionary game（演绎博弈论）

选择 MMDs作为中继节点的原因：

​ 1.MMDs从多接入网络中聚合带宽

​ 2.MMDs能够从不同的网络中中继信息

解决的问题：

​ 1.从MMDs的角度，服务更多的OMDs意味着更多的能量消耗，为了节省能量，自私的MMD会拒绝中继信息。

​ 2.MMDs为了从OMDs处获得更多的利润相互竞争，但是MMDs之间的竞争会造成MMDs的效益损失（要求的price越来越低），因此需要协调MMDs之间的带宽租赁

​ 3.OMDs的数量通常多于MMDs，每一个源节点（OMDs）想要选择最优的中继来最大化性能，当选择低效率的中继时可能会导致网络拥塞进而降低CD2D网络的性能。

相关工作

1.非激励机制

尽管那些工作能够增强频谱效率增加传输能力，但是不能解决复杂移动用户之间的资源共享问题

2.激励机制

Stackelberg game 中继选择和功率控制 但是只有一个源-目的节点对，不能直接应用到MH-CD2D网络（多个MMDs 作为中继）

系统模型
OMDs之间的演绎

所有设备半双工

MMDs租赁部分频谱给OMDs

OMD之间相互交流，平均效益通过MMDs发送给组中的

首先组中每一个OMD随机选择一个MMD中继

那么每一个MMD能够通过分配的带宽和价格估算它的效益

在与组中的其他用户交换信息之后，每一个OMD会得到效益和选择的信息，然后估算组中的平均效益

每一个OMD会选择提供更高效益的MMD，然后重新估算效益直到演绎稳定。

MMDs之间的竞争过程

每一个MMD学习并对改变作出自己的决定

当提供的带宽和售价更新时，会广播给OMD

当OMD接收到数据时，每一个OMD会调整自己的策略。

结论

该方案能够增加该MCD2D网络的性能
